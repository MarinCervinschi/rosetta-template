## The four kinds of documentation

Every page in a Rosetta site fits into exactly one of the four [Diátaxis](https://diataxis.fr/) categories. Pick the one that matches the *reader's state of mind*, not the *topic*.

<CardGrid>
	<Card title="Tutorials" icon="rocket">
		Learning-oriented. The reader wants to build their first working thing. Linear, hand-holding, concrete endpoint.
	</Card>
	<Card title="How-to guides" icon="puzzle">
		Problem-oriented. The reader already has a goal and just needs the recipe. Direct, assumes competence.
	</Card>
	<Card title="Reference" icon="list-format">
		Information-oriented. The reader is doing a lookup. Exhaustive, dry, structured.
	</Card>
	<Card title="Explanation" icon="information">
		Understanding-oriented. The reader is asking *why*. Reflective, contextual, optional.
	</Card>
</CardGrid>
