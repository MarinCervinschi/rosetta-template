# Documentation that reads like documentation.

<p class="rosetta-lede">
	Rosetta.md is a pre-configured Astro Starlight template for teams that want an AI agent to author, maintain, and query their docs without inventing a house style. It is not a framework or a hosting service — it is a baseline you clone and run.
</p>

## The four kinds of page

Every page in a Rosetta site is exactly one of these. Pick by the reader's state of mind, not by topic.

<div class="rosetta-cards">
	<a href="/tutorials/" class="rosetta-card">
		<span class="rosetta-card__label">Tutorials</span>
		<span class="rosetta-card__desc">Learning-oriented. A beginner follows steps from zero to their first working result.</span>
	</a>
	<a href="/how-to/" class="rosetta-card">
		<span class="rosetta-card__label">How-to guides</span>
		<span class="rosetta-card__desc">Problem-oriented. A competent reader has a named goal and just needs the recipe.</span>
	</a>
	<a href="/reference/" class="rosetta-card">
		<span class="rosetta-card__label">Reference</span>
		<span class="rosetta-card__desc">Information-oriented. Dry, structured, exhaustive — a map of the machinery.</span>
	</a>
	<a href="/explanation/" class="rosetta-card">
		<span class="rosetta-card__label">Explanation</span>
		<span class="rosetta-card__desc">Understanding-oriented. The why — context, trade-offs, and the shape of the problem.</span>
	</a>
</div>

## Read the rules

Agents and humans share the same contract. The canonical rules live in <code>agent-docs-rules.md</code> at the repo root — what good frontmatter looks like, when to reach for each component, the decision tree that routes a page to its Diátaxis folder.
